Instructions for submission of advanced queries
Description of OWL-S Profile

WS-HAMMER allows users to describe the capabilities they are expecting in sought resources by creating OWL Profiles. These these profiles are submitted to hammer's engine as semantic queries.
The current version of WS-HAMMER builds on the OWL-S (Semantic Markup for Web Services), hiding the user from the layered approach to markup language development (XML, RDF, OWL, and OWL-S). The following descriptions cover the relevant aspects of each profile.

Service Information
These properties of the profile provide human-readable information. A profile may have at most one service name and text description.
Service Name : Name of the sought resource.
Description : A brief description of the resource. It summarizes what the resource offers, it describes what the resource requires to work, and it indicates any additional information that the developer of the resource wants to share with the users.
Contact Information
This section provides a mechanism of refering to humans or individuals responsible for the resource (or some aspect of the resource). The range of this property is unspecified withing OWL-S, but can be restricted to some other ontology, such as the Actor ontology.
Name : Name of resource provider
Title : Title of resource provider
Phone : Telephone number
Email : Email address
Fax : Fax Number
Address : Postal (or physical) address in free format
Web URL : URL to the provider's web site
: Adds a new slot for Contact Information
: Deletes the last slot from the Contact Information
Categories
The Categories section characterize of resources on the bases of some classification that may be outside OWL-S (and possibly outside OWL). The Category is represent in OWL-S Ontology by four properties. OWL-S also defines links to well know service classification systems, like NAISC and UNSPSC. If you would like classify your service using these classificaition, or VL-e classificaition, users can directly click on the corresponding add button.
Name : The name of the actual category, which could be just a literal, or perhaps the URI of the process parameter (a property).
Taxonomy : Reference to the taxonomy scheme. It can be either a URI of the taxonomy, or a URL where the taxonomy resides, or the name of the taxonomy or anything else
Value : The value in a specific taxonomy
Code : The code associated to a taxonomy
: Adds a new Category Slot
: Deletes the last Category Slot
: Adds a new VL-e Category Slot
: Adds a new NAISC Category Slot
: Adds a new UNSPSC Category Slot
Ontology (imports)
While defining the capabilities of the sought resource, the users will be using either their OWL class definition or third parties definition. The URLs which contains these class defintions should be imported by the hammer's engine to query process.
Ontology URL : Reference to the OWL ontology. It is the URI where the ontology resides.
: Adds a new URL slot
: Deletes the last URL slot
Input ports
This section regards over instances of Inputs as defined in the Process Ontology
Input Name : Name of the input parameter
Input Type : Type of the input parameter (OWL Class, i.e. URI)
: Adds a new Input Slot
: Deletes the last Input Slot
Output ports
This section regards over instances of Outputs as defined in the Process Ontology
Output Name : Name of the output parameter
Output Type : Type of the input parameter (OWL Class, i.e. URI)
: Adds an new Output Slot
: Deletes the last Output Slot
Parameters
The Parameters section is an expandable list of properties that may accompany a profile description
Parameter Name : The name of the actual parameter, which could be just a literal, or perhaps the URI of the process parameter (a property).
sParameter : The value of the parameter within some OWL ontology.
: Adds a new Parameter Slot
: Deletes the last Parameter Slot
