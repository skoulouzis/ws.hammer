Publishing: advertisement of resources

Advertisement is a key activity of the resource model, since a provider can use this activity to inform interested parties, within the WS-VLAM community, of the existence of a specific service or component. In order to allow developers and other users to interact with a specific resource, the provider must publish the description of the resource in HAMMER. The publishing refers to task of advertising the resource's capabilities to the broker. The broker structures the service information in a repository and exposes the information to all requesters.

To allow discovery services to deal naturally with dynamic service availability, the advertisement must be periodically refreshed. For instance, when a resource is unavailable, its advertisement should also be revoked. The resource provider (or the resource itself) should send a proper announcement of the unavailability to HAMMER, in order to effectively declare that the resource is no longer available.

For more information, please refer to "How does it work?" page.
